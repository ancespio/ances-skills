# 日记模板

本文件是可复制的脱敏日记模板，不包含任何用户真实信息、私人路径、凭据或具体项目内容。

使用时，将模板复制到知识库的 `context/diary/YYYY-MM-DD_Diary.md`；若作为网页端规则与模板入口，可复制为 `context/DIARY_GUIDE.md`，并设置 `remote_access: always`。

## 模板

```markdown
---
type: context-diary
date: YYYY-MM-DD
updated: YYYY-MM-DD
remote_access: on-demand
---

# YYYY-MM-DD

关联： [相关日记](YYYY-MM-DD_Diary.md)；[相关项目画像](../persona/Project_<slug>.md)。

## 今日摘要

- [只写本次对话中确认的阶段性结果]

## 已确认事实

- [用户明确陈述的事实]
- [已完成且可验证的操作]

## 问题与处理

- 问题：[用户确认的问题]
- 处理：[已经采取的处理]
- 验证：[用户确认或可观察到的结果]

## 决策与状态变化

- 决策：[用户明确作出的决定]
- 状态变化：[项目或工作状态的已确认变化]

## 关联记录

- [相关日期日记](YYYY-MM-DD_Diary.md)
- [相关项目画像](../persona/Project_<slug>.md)

## 阻塞与下一步

- 阻塞：[用户确认的阻塞]
- 下一步：[用户确认的下一步]

## 可复用经验

- [从已发生事实中提炼的操作经验，不写未经确认的推断]

Codex Win端
```

## 写入边界

- 只记录用户明确陈述的事实、用户明确作出的决定，以及当前交互中明确表达或确认的决策倾向。
- 不写 Agent 推断、观察、心理分析、猜测或未确认的长期特征。
- 没有事实、决定或状态变化时，不创建空日记。
- 不记录真实身份号码、密钥、token、联系方式等敏感值，也不写部分掩码或占位符。
- 跨日期事件使用相对 Markdown 链接；模板中的 `<slug>` 仅是占位符，复制时替换为实际文件名。
